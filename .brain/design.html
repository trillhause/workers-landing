Workers Landing Page Design System

A comprehensive guide to the design aesthetic and patterns used in the Workers landing page. This system features a dark, technical aesthetic with clean typography, subtle animations, and glassmorphism effects optimized for a SaaS/developer tool product.

Foundation

Design Philosophy

The Workers landing page employs a modern, dark technical aesthetic characterized by:

  • High contrast: Deep blacks (#000000) paired with bright whites for maximum clarity
  • Neutral CTAs: Primary call-to-action buttons use neutral/white colors (bg-neutral-100) for clean, professional emphasis
  • Subtle orange accents: Orange reserved for pulse indicators, section markers, and subtle highlightsâ€”NOT for buttons
  • Glassmorphism effects: Backdrop blur with semi-transparent backgrounds for modern depth
  • Grid patterns and gradients: Subtle technical backgrounds that add visual interest without distraction
  • Purposeful animations: Smooth transitions and pulse effects that enhance UX
  • Semantic status colors: Cyan (monitoring), green (success), yellow (warning), red (error)
  • Typography hierarchy: Serif for hero impact, sans-serif for UI, monospace for technical labels

Technology Stack

  • Framework: Next.js with React
  • Styling: Tailwind CSS v4 with @tailwindcss/postcss
  • Fonts: Geist Sans, Geist Mono (Google Fonts)
  • Approach: Utility-first CSS with inline theme configuration

Opacity Patterns

Common opacity values used with background colors for layering and depth:

  • /10 - Very subtle backgrounds (10% opacity)
  • /20 - Subtle hover states and overlays (20% opacity)
  • /30 - Light overlays (30% opacity)
  • /40 - Medium backgrounds (40% opacity)
  • /50 - Half-transparent overlays (50% opacity)
  • /90 - Nearly opaque modals (90% opacity)

Example: bg-zinc-900/30 creates a 30% transparent zinc-900 background.

Responsive Design

Mobile-first approach with breakpoints:

  • sm: - 640px and up
  • md: - 768px and up (tablets)
  • lg: - 1024px and up (desktops)

Example: text-6xl md:text-7xl lg:text-7xl

Dark Mode

The design is dark-mode first. Light mode variables are defined but dark mode is the primary aesthetic:

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

Best Practices

  • Use semantic color names (orange for CTA, green for success)
  • Maintain consistent spacing with gap-* utilities
  • Apply transition-colors or transition-all for smooth interactions
  • Use backdrop-blur with semi-transparent backgrounds for depth
  • Keep animations subtle and purposeful
  • Leverage the zinc scale for all grayscale needs
  • Use rounded-sm for interactive elements (buttons, inputs)
  • Use rounded-lg or higher for container elements
